# Prose Structure

Use this reference to keep docs consistent and useful as agent-readable
reference material.

## House Style Pass

- Read two or three nearby pages before writing new prose.
- Preserve heading case, front matter keys, callout style, code fence language,
  wrapping, and link conventions already used by the site.
- If local docs instructions specify wrap width or heading rules, follow them.
- Prefer active, direct sentences and concrete nouns.
- Avoid marketing claims unless the page provides evidence or a clear baseline.

## Agent-Readable Shape

- Put purpose, scope, and routing cues near the top.
- Keep rules and requirements in stable sections.
- Move long examples and variants to linked pages instead of repeating them.
- Distinguish hard requirements from recommendations.
- State source ownership and validation commands explicitly.
- Use file paths, command names, schema ids, and config keys rather than vague
  references.

## Section Pattern

Use this pattern for most durable docs sections:

1. Heading.
2. Short paragraph explaining why the section exists.
3. Bullets, steps, code, table, or diagram.
4. Verification, source link, or next action when the section changes behavior.

## Editing Pass

- Remove paragraphs that do not help the reader decide, act, or understand.
- Replace unsupported certainty with source-backed claims or explicit
  assumptions.
- Collapse duplicated explanations into one owner and link to it.
- Check the page by skimming only headings, tables, captions, and code blocks.
